DFTFringe Version History

  • Version v1.0 Initial release
  • V1.1-beta corrected saving, inport and export of Mirror dialog with OpenFringe
  • v1.2-beta
    • Added credits to about
    • Reduced size of test stand wizard and made it more resizable
    • Save mirror outlines for loaded interferogram every time the "Done" or "make surface" buttons are used.
      Look for saved outline file associated with the loading interferogram and use it if found.
    • Internal change to how zernike values are computed. Now using Polar definitions.
    • Increased zernike terms to 48 using all 6th order terms
    • Added batch processing of interferograms both manual assist and auto mode.
    • Added enable all and enable spherical only buttons to Metrics display
    • Added interferogram wavelength to metrics
    • Added average of stand/system induced terms to stand astig reporting
  • Version 1.3-beta
    • Added hot keys for editing the outline.
    • Changed outline help to a button to open a docked help window on the left side.
    • Corrected and reworked logic to save wavefront stats. Moved display of stats to under the View Menu
    • Added tool to jitter size of outline and save wavefronts. This helps determine if the outline is correct. Will enhance it next version.
  • Version 1.4-beta
    • Added outline helper to shift outlines in auto mode.
    • Additional fixes to wavefront stats.
    • Added selectable background color to 3D display.
    • Added show all 3D wavefronts.
    • Corrected bug in writing outlines for center hole outline.
  • Version 1.5-beta
    • Added full sized popup windows to display 3D and contour displays using right click menus
  • Version 1.6-beta
    • Corrected zernike calculating bug introduced in vers 1.2 Should give reliable results once again.
    • Added software null error tolerance in view menu.
    • Modified outline boundary slightly to eliminate fringes around mirror after DFT analysis.
    • Added gamma (contrast) horizontal slider control to DFT tools.
    • Corrected issues when several wavefronts are loaded and a change propigates thru all with the last one in the list displaying the correct metrics.Hopefully smoothing remains if enabled.
    • Gaussian smoothing modified to not include data outise mirror outside board when calculating smoothing just inside the outline.
  • Version 1.6A-beta
    • Bug fix for star test crash
    • Bug fix to stop outline from changing on save and restore operations
    • Bug fix for zernike values computation.
  • Version 1.7-beta
    • Added cc and null values to report and metrics.
    • Corrected loading of saved center outline.
    • Change how igram outline is zoomed into including logic for process center outline.
    • Added warning to averaging of wavefronts if any have different signs for primary SA
    • Corrected save and restore of Flip igram horizontal. Deleted it from Preferences and kept it in the mirror config.
    • changed Metric zernike term values to be read only.
    • Added loaded igram to report
  • Version 1.8-beta
    • Only update selected wavefront/s
    • Added zernike based editing save and load
    • Added Foucault and ronchi simulation
    • Fixed bug in display of ROC in Mirror dialog when changing F number
    • Fixed metrics and report display of Zernike terms RMS value.
    • Fixed test stand removal when average files are a different size. Also fixed the astig report so that text does not overlap in the contour plot footers.
  • Version 1.9-beta
    • Added elliptical Flat processing
    • Changed zernike rms values to use sign of term instead of all being positive
    • Enabled translation support of most text strings
    • Modified igram and wavefront simulation to use the full zernike set and added import from wavefront button.

Known bugs

Outline does not always appear on interferogram when processing in batch mode. Workaround is to rotate the mouse wheel to resize it when in Manual mode batch.
